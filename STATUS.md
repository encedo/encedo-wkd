## nginx Architecture

Carbonio nginx (`/opt/zextras/common/sbin/nginx`):

```
nginx.conf.web.https       ← our file (generated by inject.sh)
  server { listen 65.21.170.222:443 ssl; server_name openpgpkey.*; }
  → proxy_pass http://127.0.0.1:8089

nginx.conf.web.https.default  ← Carbonio default
  server { listen 65.21.170.222:443 default_server; server_name mailserver.*; }
```

**Key point:** `listen 65.21.170.222:443` (not `*:443`) — must use the same socket
as Carbonio's `default_server`, otherwise SNI routing does not work.

## Publish API (via main Carbonio domain)

```
POST https://mailserver.encedo.com/wkd/api/publish
X-Auth-Token: <carbonio_session_token>

nginx extensions/backend-wkd.conf → location /wkd/ → proxy → 127.0.0.1:8089
server.py → validates token via Carbonio SOAP GetInfoRequest
→ stores key in /var/encedo-wkd/cache/<domain>/<wkd_hash>
```

nginx extensions are installed by `install.sh` (step [4/6]) and live at:
- `/opt/zextras/conf/nginx/extensions/upstream-wkd.conf`
- `/opt/zextras/conf/nginx/extensions/backend-wkd.conf`

## Re-deployment (after Carbonio upgrade or adding a new domain)

```bash
sudo ./encedo-wkd-nginx-inject.sh
```

The script automatically:
1. Fetches domain list from Carbonio (`zmprov gad`)
2. Checks DNS CNAME for each `openpgpkey.*` subdomain
3. Requests Let's Encrypt certificates for new domains (if missing)
4. Detects server IP from `nginx.conf.web.https.default`
5. Generates nginx config and reloads nginx
6. Generates `config.json`
